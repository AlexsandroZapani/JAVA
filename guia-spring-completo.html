Etapa 1 de 10 · Instalação
Instalar o Java (JDK 17)
A linguagem em si. Sem Java instalado, nada roda.
💡 O que é o JDK?
🧠 Conceito — Java e o JDK

Java é uma linguagem de programação. Quando você escreve código Java, o computador não entende diretamente — ele precisa de um "tradutor".

O JDK (Java Development Kit = Kit de Desenvolvimento Java) é esse tradutor. Ele tem tudo que você precisa para escrever, compilar e executar programas Java.

A versão 17 é a LTS (Long Term Support) — ou seja, é mantida por muitos anos e é a recomendada para projetos sérios.

🪟 Instalação no Windows
1
Baixe o instalador

Acesse adoptium.net no navegador. Clique em "Latest LTS Release" e depois baixe o .msi para Windows.

2
Execute o instalador

Abra o arquivo .msi baixado. Clique em "Next" em tudo. Na tela que mostrar opções, marque a opção "Set JAVA_HOME variable" se aparecer.

3
Verificar a instalação

Abra o Prompt de Comando (Win+R → digite cmd → Enter) e execute:

 Prompt de Comando (cmd)
C:\> java -version
openjdk version "17.0.9" 2023-10-17
OpenJDK Runtime Environment Temurin-17...
# Se aparecer algo parecido acima, está instalado! ✅
# Se aparecer erro, reinicie o computador e tente novamente.
🍎 Instalação no macOS
✅ Método recomendado — Homebrew

O Homebrew é um gerenciador de pacotes para Mac. Se não tiver, instale primeiro com o comando abaixo, depois instale o Java.

 Terminal (Mac)
# Passo 1: Instalar o Homebrew (se não tiver)
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# Passo 2: Instalar o Java 17
$ brew install --cask temurin@17
# Passo 3: Verificar
$ java -version
openjdk version "17.0.9" ...
🐧 Instalação no Linux (Ubuntu/Debian)
 Terminal (Linux)
# Atualizar lista de pacotes
$ sudo apt update
# Instalar Java 17
$ sudo apt install openjdk-17-jdk -y
# Verificar
$ java -version
openjdk version "17.0.9" ...
⚠️ Se o comando "java" não for reconhecido

No Windows, pode ser necessário adicionar o Java ao PATH do sistema. O instalador Temurin faz isso automaticamente. Se não funcionar, reinicie o computador e tente de novo. Se ainda não funcionar, pesquise "como adicionar java ao path windows 10".

Etapa 2 de 10 · Instalação
Instalar o Maven
O gerenciador que baixa as dependências do projeto automaticamente.
🧠 Conceito — O que é Maven?

Imagine que você vai cozinhar uma receita e ela pede 15 ingredientes. Você teria que ir ao mercado, comprar cada um, trazer, organizar...

O Maven faz exatamente isso com bibliotecas Java. Você lista no arquivo pom.xml tudo que precisa (Spring Boot, H2, Lombok...) e o Maven baixa tudo da internet automaticamente. Sem Maven, você teria que baixar cada biblioteca manualmente e adicionar ao projeto.

🪟 Windows
1
Baixar

Acesse maven.apache.org/download.cgi e baixe o arquivo apache-maven-3.x.x-bin.zip.

2
Extrair

Extraia o ZIP para uma pasta simples, por exemplo: C:\maven

3
Adicionar ao PATH

Pesquise "variáveis de ambiente" no Windows → clique em "Editar variáveis do sistema" → "Variáveis de ambiente" → em "Path" clique "Editar" → "Novo" → adicione: C:\maven\bin

4
Verificar

Feche e reabra o Prompt de Comando. Execute:

 cmd
C:\> mvn -version
Apache Maven 3.9.x
Maven home: C:\maven
Java version: 17.0.9 ...
🍎 macOS / 🐧 Linux
 Terminal
# macOS (via Homebrew)
$ brew install maven
# Linux (Ubuntu/Debian)
$ sudo apt install maven -y
# Verificar nos dois sistemas
$ mvn -version
Apache Maven 3.x.x ...

🧠 Conceito — O que é uma IDE?

IDE (Integrated Development Environment) é um editor de código inteligente. Não é um bloco de notas — a IDE entende Java, sugere completações, aponta erros enquanto você digita, organiza os arquivos, roda o projeto com um clique.

O IntelliJ IDEA Community é gratuito, e é a melhor IDE para Java no mercado.

Etapa 4 de 10 · Projeto
Criar o Projeto Spring Boot
Gerar a estrutura base e entender cada arquivo criado.
🧠 Conceito — O que é Spring Boot?

Spring é um framework (conjunto de ferramentas) para Java que facilita muito criar aplicações. Spring Boot é a versão "autoconfigurada" — ele toma decisões por você e você escreve muito menos código para ter tudo funcionando.

Antes do Spring Boot, configurar um servidor Java levava dias. Com Spring Boot, em minutos você tem um servidor rodando.

🌐 Opção A: Spring Initializr (recomendado)
1
Acesse start.spring.io

Este site gera a estrutura do projeto automaticamente.

2
Configure assim:

Project: Maven · Language: Java · Spring Boot: 3.2.x
Group: com.projeto · Artifact: biblioteca · Java: 17

3
Adicionar dependências

Clique em "ADD DEPENDENCIES" e busque/adicione: Spring Web, Spring Data JPA, H2 Database, Validation, Lombok.

4
GENERATE → Extrair → Abrir no IntelliJ

Clique em GENERATE, um ZIP vai baixar. Extraia em uma pasta (ex: C:\projetos\biblioteca) e abra no IntelliJ: File → Open → selecionar a pasta.

📄 O pom.xml explicado
🧠 Conceito — pom.xml

O pom.xml (Project Object Model) é o arquivo que diz ao Maven quem é o seu projeto e do que ele depende. É como uma lista de compras — você lista o que precisa, o Maven busca.

biblioteca/pom.xml
<!-- O pom.xml fica na RAIZ do projeto (pasta biblioteca/) -->
<project xmlns="http://maven.apache.org/POM/4.0.0">
  <modelVersion>4.0.0</modelVersion>

  <!--
    PARENT: herdamos tudo do Spring Boot.
    Ele já definiu versões testadas de todas as bibliotecas.
    Não precisamos nos preocupar com conflitos de versão.
  -->
  <parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
    <relativePath/>
  </parent>

  <!-- Identidade do projeto -->
  <groupId>com.projeto</groupId>     <!-- sua "empresa" -->
  <artifactId>biblioteca</artifactId> <!-- nome do projeto -->
  <version>0.0.1-SNAPSHOT</version>  <!-- versão atual -->
  <packaging>jar</packaging>         <!-- empacota como .jar executável -->

  <properties>
    <java.version>17</java.version> <!-- usamos Java 17 -->
  </properties>

  <dependencies>

    <!--
      SPRING WEB
      Dá tudo para criar uma API REST:
      - Servidor HTTP embutido (Tomcat) — sem instalar nada
      - @RestController, @GetMapping, @PostMapping...
      - Conversão automática Java ↔ JSON
    -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
    </dependency>

    <!--
      SPRING DATA JPA
      JPA = Java Persistence API
      Permite trabalhar com banco de dados usando objetos Java.
      Você cria uma classe Java → ela vira uma tabela no banco.
      Hibernate é a implementação por baixo.
    -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-data-jpa</artifactId>
    </dependency>

    <!--
      VALIDATION
      Permite validar dados recebidos com anotações simples:
      @NotBlank — campo não pode ser vazio
      @Size     — tamanho mínimo/máximo
      @Min/@Max — valor mínimo/máximo para números
    -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-validation</artifactId>
    </dependency>

    <!--
      H2 DATABASE
      Banco de dados que roda DENTRO da aplicação, na memória RAM.
      Não precisa instalar nada separado.
      Dados somem ao reiniciar (perfeito para aprender!).
      scope=runtime: só usado quando a app está rodando.
    -->
    <dependency>
      <groupId>com.h2database</groupId>
      <artifactId>h2</artifactId>
      <scope>runtime</scope>
    </dependency>

    <!--
      LOMBOK
      Gera código repetitivo automaticamente. Sem ele:
      - Para cada campo, você escreveria getNome(), setNome()...
      - Com Lombok, só coloca @Getter @Setter na classe.
      optional=true: só usado na compilação, não no jar final.
    -->
    <dependency>
      <groupId>org.projectlombok</groupId>
      <artifactId>lombok</artifactId>
      <optional>true</optional>
    </dependency>

    <!-- TESTES: JUnit 5 + Mockito (só usado nos testes) -->
    <dependency>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-test</artifactId>
      <scope>test</scope>
    </dependency>

  </dependencies>

  <build>
    <plugins>
      <!-- Plugin que cria o JAR executável final -->
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
      </plugin>
    </plugins>
  </build>
</project>

Etapa 5 de 10 · Configuração
Banco de Dados + application.properties
Configurar o H2 e criar os dados iniciais de exemplo.
🧠 Conceito — O que é o H2 e o application.properties?

O H2 é um banco de dados que roda completamente dentro da sua aplicação Java, na memória RAM. Você não precisa instalar MySQL, PostgreSQL nem nada separado. Ao iniciar a app, o banco cria as tabelas; ao desligar, tudo some. Perfeito para aprender!

O application.properties é o arquivo de configuração do Spring Boot. Ele fica em src/main/resources/ e o Spring lê ele ao iniciar para saber: em qual porta rodar, como conectar ao banco, que nível de log mostrar, etc.

📄 Crie: src/main/resources/application.properties
src/main/resources/application.properties
# ─────────────────────────────────────────────────────────────
# NOME E PORTA
# A API vai rodar em: http://localhost:8080
# ─────────────────────────────────────────────────────────────
spring.application.name=biblioteca
server.port=8080

# ─────────────────────────────────────────────────────────────
# BANCO DE DADOS H2
#
# jdbc:h2:mem:biblioteca = banco em MEMÓRIA chamado "biblioteca"
#   mem: = in-memory (na RAM, sem arquivo em disco)
#   MODE=MySQL = simula comportamento do MySQL
#   DB_CLOSE_DELAY=-1 = mantém o banco vivo enquanto app roda
# ─────────────────────────────────────────────────────────────
spring.datasource.url=jdbc:h2:mem:biblioteca;MODE=MySQL;DB_CLOSE_DELAY=-1
spring.datasource.driver-class-name=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# ─────────────────────────────────────────────────────────────
# CONSOLE WEB DO H2
#
# Habilita uma interface visual para ver o banco de dados!
# Acesse: http://localhost:8080/h2-console
# JDBC URL: jdbc:h2:mem:biblioteca | User: sa | Senha: (vazio)
# ─────────────────────────────────────────────────────────────
spring.h2.console.enabled=true
spring.h2.console.path=/h2-console

# ─────────────────────────────────────────────────────────────
# JPA / HIBERNATE
#
# ddl-auto define o que o Hibernate faz com o banco ao iniciar:
#   create-drop = APAGA e RECRIA tabelas a cada reinício (ótimo para dev!)
#   update      = mantém dados, só atualiza estrutura se mudou
#   none        = não faz nada automaticamente (produção)
#
# show-sql = mostra no console cada SQL gerado (ótimo para aprender!)
# format_sql = formata o SQL para ser legível
# ─────────────────────────────────────────────────────────────
spring.jpa.hibernate.ddl-auto=create-drop
spring.jpa.database-platform=org.hibernate.dialect.H2Dialect
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true

# ─────────────────────────────────────────────────────────────
# DADOS INICIAIS
#
# spring.sql.init.mode=always = executa o data.sql ao iniciar
# defer-datasource = garante que as tabelas existam ANTES do data.sql
# ─────────────────────────────────────────────────────────────
spring.sql.init.mode=always
spring.jpa.defer-datasource-initialization=true

📄 BibliotecaApplication.java — Ponto de entrada
🧠 Conceito — O método main()

Todo programa Java começa pelo método main(). O Spring Boot usa esse método para iniciar tudo: banco, servidor, controllers... Um único método, toda a aplicação sobe.

com/projeto/biblioteca/BibliotecaApplication.java
package com.projeto.biblioteca;
// "package" = pasta lógica. Todo arquivo Java começa declarando
// o pacote (pasta) onde ele mora. Isso evita conflitos de nomes.

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
// "import" = importar classes de outros lugares para usar aqui

@SpringBootApplication
// Esta anotação faz 3 coisas ao mesmo tempo:
// 1. Diz que esta classe pode declarar configurações (beans)
// 2. Ativa a auto-configuração do Spring (Spring decide tudo sozinho)
// 3. Diz para o Spring varrer todos os subpacotes procurando
//    classes anotadas com @Service, @Repository, @Controller...
public class BibliotecaApplication {

  public static void main(String[] args) {
    // Este é o ponto de entrada do programa.
    // SpringApplication.run() inicializa tudo:
    //  - Conecta ao banco H2
    //  - Cria as tabelas (via Hibernate)
    //  - Executa data.sql
    //  - Registra todos os endpoints HTTP
    //  - Inicia o servidor Tomcat na porta 8080
    SpringApplication.run(BibliotecaApplication.class, args);
    System.out.println("✅ API rodando em: http://localhost:8080");
    System.out.println("🗄️  H2 Console:    http://localhost:8080/h2-console");
  }
}

Etapa 6 de 10 · Código
Model — Entidades JPA
As classes Java que se tornam tabelas no banco de dados.
🧠 Conceito — O que é uma Entidade?

Uma entidade é uma classe Java que representa uma tabela no banco de dados. Você anota ela com @Entity e o Hibernate (por baixo do Spring) cria a tabela automaticamente.

Cada campo da classe = uma coluna na tabela.
Cada objeto criado = uma linha na tabela.

O relacionamento: um Autor tem muitos Livros. No banco, isso vira uma coluna autor_id na tabela livros (chave estrangeira).

📄 Crie: model/Autor.java
model/Autor.java
package com.projeto.biblioteca.model;

import jakarta.persistence.*;
import jakarta.validation.constraints.*;
import lombok.*;
import java.time.LocalDate;
import java.util.ArrayList;
import java.util.List;

// ─── ANOTAÇÕES LOMBOK ──────────────────────────────────────
// Lombok gera código repetitivo para nós automaticamente.
// Sem Lombok, para cada campo você escreveria:
//   public String getNome() { return nome; }
//   public void setNome(String nome) { this.nome = nome; }
// Com Lombok, só coloca @Getter @Setter e ele faz isso!

@Getter          // Gera: getNome(), getNacionalidade(), getId()...
@Setter          // Gera: setNome(), setNacionalidade()...
@Builder         // Permite: Autor.builder().nome("x").build()
@NoArgsConstructor  // Gera: new Autor() — construtor sem argumentos
@AllArgsConstructor // Gera: new Autor(id, nome, ...) — com todos os campos
@ToString           // Gera: toString() para imprimir o objeto

// ─── ANOTAÇÕES JPA ────────────────────────────────────────
@Entity
// @Entity = esta classe é uma tabela no banco de dados.
// O Hibernate vai criar: CREATE TABLE autores (...)

@Table(name = "autores")
// @Table = define o nome da tabela no banco.
// Sem isso, o nome seria "Autor" (nome da classe).

public class Autor {

  // ─── CHAVE PRIMÁRIA ───────────────────────────────────────
  // Todo registro num banco precisa de um identificador único.
  // O banco vai gerar automaticamente: 1, 2, 3, 4...

  @Id
  // @Id = este campo é a chave primária (Primary Key).
  // No banco: coluna "id" com tipo BIGINT PRIMARY KEY.

  @GeneratedValue(strategy = GenerationType.IDENTITY)
  // @GeneratedValue = o banco gera o valor automaticamente.
  // IDENTITY = auto-incremento: 1, 2, 3, 4...
  // Você nunca precisa informar o ID ao criar um autor.

  private Long id;
  // Long = número inteiro longo (pode chegar em bilhões).
  // Perfeito para IDs.

  // ─── CAMPOS DA TABELA ─────────────────────────────────────
  // Cada campo aqui = uma coluna na tabela "autores"

  @NotBlank(message = "Nome é obrigatório")
  // @NotBlank = validação: o campo não pode ser:
  //   - null (sem valor)
  //   - "" (string vazia)
  //   - "   " (só espaços)
  // A mensagem aparece quando a validação falha.

  @Size(min = 2, max = 100, message = "Nome: 2 a 100 caracteres")
  // @Size = tamanho mínimo e máximo da string

  @Column(name = "nome", nullable = false, length = 100)
  // @Column = define como a coluna aparece no banco:
  //   name = "nome"   → nome da coluna no banco
  //   nullable = false → NOT NULL no SQL (obrigatório)
  //   length = 100    → VARCHAR(100)

  private String nome;

  @NotBlank(message = "Nacionalidade é obrigatória")
  @Column(name = "nacionalidade", nullable = false, length = 50)
  private String nacionalidade;

  @Column(name = "data_nascimento")
  // LocalDate = representa apenas uma data (sem hora): ex: 1839-06-21
  // Sem @NotNull aqui: data de nascimento é opcional
  private LocalDate dataNascimento;

  // ─── RELACIONAMENTO 1:N ───────────────────────────────────
  //
  // Um Autor tem MUITOS Livros → @OneToMany
  // Um Livro tem UM Autor    → @ManyToOne (na classe Livro)
  //
  // mappedBy = "autor"
  //   → diz que a classe Livro é "dona" do relacionamento
  //   → o campo "autor" dentro de Livro tem a chave estrangeira
  //
  // cascade = CascadeType.ALL
  //   → ao deletar um autor, seus livros também são deletados
  //
  // fetch = FetchType.LAZY
  //   → os livros SÓ são carregados do banco quando pedirmos
  //   → economiza memória: não carrega livros desnecessariamente
  //
  // @ToString.Exclude
  //   → sem isso, toString() de Autor chamaria toString() de Livro
  //   que chamaria toString() de Autor... loop infinito! 💥

  @OneToMany(mappedBy = "autor", cascade = CascadeType.ALL, fetch = FetchType.LAZY)
  @ToString.Exclude
  private List<Livro> livros = new ArrayList<>();
  // List = lista de livros deste autor
  // Inicializamos como ArrayList vazio para não dar NullPointerException
}

import jakarta.persistence.*;
import jakarta.validation.constraints.*;
import lombok.*;
import java.math.BigDecimal;

@Getter @Setter @Builder @ToString
@NoArgsConstructor @AllArgsConstructor
@Entity
@Table(name = "livros")
public class Livro {

  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  @NotBlank(message = "Título é obrigatório")
  @Size(min = 1, max = 200)
  @Column(name = "titulo", nullable = false)
  private String titulo;

  // ISBN = International Standard Book Number
  // Código único mundial que identifica um livro.
  // unique = true → o banco não permite dois livros com o mesmo ISBN
  @NotBlank(message = "ISBN é obrigatório")
  @Column(name = "isbn", nullable = false, unique = true)
  private String isbn;

  @NotNull(message = "Ano de publicação é obrigatório")
  // @NotNull = campo não pode ser null (para tipos não-String, use @NotNull)
  // @NotBlank é só para String. Para Integer, Long, BigDecimal → @NotNull
  @Min(value = 1000, message = "Ano inválido")
  @Max(value = 2100, message = "Ano inválido")
  @Column(name = "ano_publicacao", nullable = false)
  private Integer anoPublicacao;

  // BigDecimal para valores monetários!
  // Por quê não double ou float?
  // double: 0.1 + 0.2 = 0.30000000000000004 (erro de ponto flutuante)
  // BigDecimal: 0.1 + 0.2 = 0.3 (preciso, sem erros de centavos)
  @NotNull(message = "Preço é obrigatório")
  @DecimalMin(value = "0.01", message = "Preço deve ser positivo")
  @Column(name = "preco", nullable = false, precision = 10, scale = 2)
  // precision=10 = máx 10 dígitos no total
  // scale=2      = 2 dígitos após a vírgula (centavos)
  private BigDecimal preco;

  @NotNull(message = "Estoque é obrigatório")
  @Min(value = 0, message = "Estoque não pode ser negativo")
  @Column(name = "estoque", nullable = false)
  private Integer estoque;

  // ─── RELACIONAMENTO N:1 ───────────────────────────────────
  //
  // @ManyToOne = Muitos Livros → Um Autor
  // Este é o lado "dono" do relacionamento.
  // Aqui fica a coluna "autor_id" no banco (chave estrangeira).
  //
  // FetchType.EAGER = carrega o autor junto com o livro.
  // Quando buscar um Livro, o Autor já vem junto automaticamente.
  //
  // @JoinColumn(name = "autor_id")
  //   → define o nome da coluna de chave estrangeira no banco
  //   → Banco cria: autor_id BIGINT REFERENCES autores(id)
  @ManyToOne(fetch = FetchType.EAGER)
  @JoinColumn(name = "autor_id", nullable = false)
  @NotNull(message = "Autor é obrigatório")
  @ToString.Exclude
  private Autor autor;
}

Repository — Acesso ao Banco
O Spring gera o SQL automaticamente a partir do nome dos métodos.
🧠 Conceito — Repository Pattern

O Repository é a camada que fala com o banco de dados. Ao criar uma interface que estende JpaRepository<Autor, Long>, o Spring cria automaticamente a implementação com dezenas de métodos prontos. Você não precisa escrever nenhuma query SQL para as operações básicas.

O que você ganha de graça: save() (INSERT/UPDATE), findById() (SELECT), findAll() (SELECT *), deleteById() (DELETE), count(), existsById()...

Query Method: Para buscas customizadas, o Spring lê o nome do método e gera o SQL. findByNomeContainingIgnoreCase("machado") → WHERE LOWER(nome) LIKE '%machado%'. Automático!

DTOs, Exceções e Services
Objetos de transferência, erros customizados e regras de negócio.
🧠 Conceito — DTO, Service e Exceções

DTO (Data Transfer Object): define exatamente o que entra e o que sai da API. Separa o que o banco guarda do que o cliente recebe — mais seguro e flexível.

Service: contém as regras de negócio. Ex: "não pode criar dois autores com o mesmo nome". Ele converte DTO ↔ Entity, chama o Repository e gerencia transações.

Exceções customizadas: quando algo dá errado (ID não encontrado, ISBN duplicado), lançamos exceções próprias. O GlobalExceptionHandler as captura e retorna JSON limpo com o status HTTP correto.

package com.projeto.biblioteca.exception;

import org.springframework.http.*;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.*;
import java.time.LocalDateTime;
import java.util.*;

@ControllerAdvice
// @ControllerAdvice = esta classe é um "interceptador" global.
// Ela captura exceções lançadas em QUALQUER controller da app.
// Sem isso, quando um erro ocorre, o Spring retornaria uma
// página de erro feia com stack trace técnico para o cliente.
public class GlobalExceptionHandler {

  // Record que representa a resposta de erro — JSON padronizado
  record ErroResponse(int status, String mensagem, LocalDateTime timestamp) {}

  @ExceptionHandler(ResourceNotFoundException.class)
  // @ExceptionHandler = quando ResourceNotFoundException for lançada,
  // este método é chamado. Retorna HTTP 404 com JSON de erro.
  public ResponseEntity<ErroResponse> handle404(ResourceNotFoundException ex) {
    return ResponseEntity
      .status(HttpStatus.NOT_FOUND) // HTTP 404
      .body(new ErroResponse(404, ex.getMessage(), LocalDateTime.now()));
  }

  @ExceptionHandler(BusinessException.class)
  public ResponseEntity<ErroResponse> handle409(BusinessException ex) {
    return ResponseEntity
      .status(HttpStatus.CONFLICT) // HTTP 409
      .body(new ErroResponse(409, ex.getMessage(), LocalDateTime.now()));
  }

  @ExceptionHandler(MethodArgumentNotValidException.class)
  // Captura erros do @Valid — quando os dados enviados não passam
  // nas validações (@NotBlank, @Size, @Min, etc.)
  public ResponseEntity<Map<String, Object>> handle400(MethodArgumentNotValidException ex) {
    Map<String, String> campos = new HashMap<>();
    for (FieldError e : ex.getBindingResult().getFieldErrors())
      campos.put(e.getField(), e.getDefaultMessage());
      // Coleta todos os campos com erro: {"nome":"Nome é obrigatório"}
    return ResponseEntity.badRequest() // HTTP 400
      .body(Map.of("status", 400, "campos", campos, "timestamp", LocalDateTime.now()));
  }
}

package com.projeto.biblioteca.service;

import com.projeto.biblioteca.dto.AutorDTO;
import com.projeto.biblioteca.dto.AutorDTO.*;
import com.projeto.biblioteca.exception.BusinessException;
import com.projeto.biblioteca.exception.ResourceNotFoundException;
import com.projeto.biblioteca.model.Autor;
import com.projeto.biblioteca.repository.AutorRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.util.List;

@Service
// @Service = marca esta classe como componente de serviço.
// O Spring a detecta, cria uma instância e a injeta onde for necessária.
// É equivalente a @Component, mas semanticamente mais claro.
public class AutorService {

  // ─── INJEÇÃO DE DEPENDÊNCIA POR CONSTRUTOR ────────────────
  //
  // "Injeção de Dependência" = o Spring nos entrega o AutorRepository
  // em vez de criarmos com "new AutorRepository()".
  //
  // Por que usar construtor em vez de @Autowired no campo?
  //   ✅ Melhor para testes (podemos passar um mock)
  //   ✅ O campo pode ser "final" (imutável após criação)
  //   ✅ Forma recomendada atualmente pelo Spring
  private final AutorRepository autorRepository;

  public AutorService(AutorRepository autorRepository) {
    this.autorRepository = autorRepository;
    // "this.autorRepository" = campo da classe
    // "autorRepository"      = parâmetro do construtor
  }

  // ══════════════════════════════════════════════════════════
  //  C — CREATE (criar)
  // ══════════════════════════════════════════════════════════
  @Transactional
  // @Transactional = tudo dentro deste método acontece em uma transação.
  // Se qualquer linha lançar exceção, o banco faz ROLLBACK (desfaz tudo).
  // Se tudo der certo, faz COMMIT (confirma as mudanças).
  public AutorResponseDTO criar(AutorRequestDTO dto) {

    // REGRA DE NEGÓCIO: nome deve ser único
    // findByNomeIgnoreCase retorna Optional
    // ifPresent = executa o código se o Optional não estiver vazio
    autorRepository.findByNomeIgnoreCase(dto.nome()).ifPresent(existente -> {
      throw new BusinessException("Já existe um autor com o nome: " + dto.nome());
    });

    Autor autor = paraEntidade(dto);            // DTO → Entity
    Autor salvo = autorRepository.save(autor);  // INSERT no banco
    return paraResponseDTO(salvo);             // Entity → DTO de resposta
  }

  // ══════════════════════════════════════════════════════════
  //  R — READ (ler)
  // ══════════════════════════════════════════════════════════
  @Transactional(readOnly = true)
  // readOnly = true → otimização para leitura:
  // O Hibernate não rastreia mudanças nos objetos (mais performance)
  public List<AutorResponseDTO> listarTodos() {
    return autorRepository.findAll()    // SELECT * FROM autores
      .stream()                          // transforma a List em Stream (processamento)
      .map(this::paraResponseDTO)        // converte cada Autor → AutorResponseDTO
      .toList();                          // junta de volta em List
  }

  @Transactional(readOnly = true)
  public AutorResponseDTO buscarPorId(Long id) {
    Autor autor = autorRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Autor", "id", id));
      // orElseThrow = retorna o Autor SE encontrado, OU lança a exceção.
      // A exceção é capturada pelo GlobalExceptionHandler → retorna 404.
    return paraResponseDTO(autor);
  }

  @Transactional(readOnly = true)
  public List<AutorResponseDTO> buscarPorNome(String nome) {
    return autorRepository.findByNomeContainingIgnoreCase(nome)
      .stream().map(this::paraResponseDTO).toList();
  }

  // ══════════════════════════════════════════════════════════
  //  U — UPDATE (atualizar)
  // ══════════════════════════════════════════════════════════
  @Transactional
  public AutorResponseDTO atualizar(Long id, AutorRequestDTO dto) {
    Autor autor = autorRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Autor", "id", id));

    // Alteramos os campos do objeto existente.
    // NÃO criamos um novo Autor — se criássemos, o save() faria INSERT
    // em vez de UPDATE. Como o objeto "autor" tem ID, faz UPDATE.
    autor.setNome(dto.nome());
    autor.setNacionalidade(dto.nacionalidade());
    autor.setDataNascimento(dto.dataNascimento());

    return paraResponseDTO(autorRepository.save(autor)); // UPDATE
  }

  // ══════════════════════════════════════════════════════════
  //  D — DELETE (deletar)
  // ══════════════════════════════════════════════════════════
  @Transactional
  public void deletar(Long id) {
    Autor autor = autorRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Autor", "id", id));

    // REGRA: não deletar autor que tem livros cadastrados
    if (!autor.getLivros().isEmpty()) {
      throw new BusinessException(
        "Não é possível deletar o autor pois ele possui " +
        autor.getLivros().size() + " livro(s). Delete os livros primeiro."
      );
    }
    autorRepository.delete(autor); // DELETE FROM autores WHERE id = ?
  }

  // ─── MÉTODOS PRIVADOS DE CONVERSÃO ────────────────────────

  // DTO de entrada → Entidade JPA
  private Autor paraEntidade(AutorRequestDTO dto) {
    return Autor.builder()        // inicia o Builder (Lombok)
      .nome(dto.nome())           // dto.nome() = getter do record
      .nacionalidade(dto.nacionalidade())
      .dataNascimento(dto.dataNascimento())
      .build();                  // cria o objeto Autor
    // Não setamos ID: o banco vai gerar automaticamente
  }

  // Entidade JPA → DTO de saída (public para o LivroService usar)
  public AutorResponseDTO paraResponseDTO(Autor a) {
    return new AutorResponseDTO(
      a.getId(), a.getNome(), a.getNacionalidade(),
      a.getDataNascimento(),
      a.getLivros() != null ? a.getLivros().size() : 0
    );
  }

  public AutorDTO.AutorResumoDTO paraResumoDTO(Autor a) {
    return new AutorDTO.AutorResumoDTO(a.getId(), a.getNome(), a.getNacionalidade());
  }
}

package com.projeto.biblioteca.service;

import com.projeto.biblioteca.dto.LivroDTO.*;
import com.projeto.biblioteca.exception.BusinessException;
import com.projeto.biblioteca.exception.ResourceNotFoundException;
import com.projeto.biblioteca.model.Autor;
import com.projeto.biblioteca.model.Livro;
import com.projeto.biblioteca.repository.AutorRepository;
import com.projeto.biblioteca.repository.LivroRepository;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;
import java.math.BigDecimal;
import java.util.List;

@Service
public class LivroService {

  // Dois repositories + AutorService (para conversão de DTO)
  private final LivroRepository  livroRepository;
  private final AutorRepository  autorRepository;
  private final AutorService     autorService;

  public LivroService(LivroRepository livroRepository,
                       AutorRepository autorRepository,
                       AutorService autorService) {
    this.livroRepository = livroRepository;
    this.autorRepository = autorRepository;
    this.autorService = autorService;
  }

  @Transactional
  public LivroResponseDTO criar(LivroRequestDTO dto) {
    // Regra: ISBN é único no mundo — não pode duplicar
    if (livroRepository.existsByIsbn(dto.isbn()))
      throw new BusinessException("Já existe um livro com o ISBN: " + dto.isbn());

    // Buscar o Autor pelo ID recebido no DTO
    Autor autor = autorRepository.findById(dto.autorId())
      .orElseThrow(() -> new ResourceNotFoundException("Autor", "id", dto.autorId()));

    Livro livro = paraEntidade(dto, autor);
    return paraResponseDTO(livroRepository.save(livro));
  }

  @Transactional(readOnly = true)
  public List<LivroResponseDTO> listarTodos() {
    return livroRepository.findAllByOrderByTituloAsc()
      .stream().map(this::paraResponseDTO).toList();
  }

  @Transactional(readOnly = true)
  public LivroResponseDTO buscarPorId(Long id) {
    return paraResponseDTO(livroRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Livro", "id", id)));
  }

  @Transactional(readOnly = true)
  public LivroResponseDTO buscarPorIsbn(String isbn) {
    return paraResponseDTO(livroRepository.findByIsbn(isbn)
      .orElseThrow(() -> new ResourceNotFoundException("Livro", "isbn", isbn)));
  }

  @Transactional(readOnly = true)
  public List<LivroResponseDTO> buscarPorTitulo(String titulo) {
    return livroRepository.findByTituloContainingIgnoreCase(titulo)
      .stream().map(this::paraResponseDTO).toList();
  }

  @Transactional(readOnly = true)
  public List<LivroResponseDTO> listarPorAutor(Long autorId) {
    if (!autorRepository.existsById(autorId))
      throw new ResourceNotFoundException("Autor", "id", autorId);
    return livroRepository.findByAutor_Id(autorId)
      .stream().map(this::paraResponseDTO).toList();
  }

  @Transactional(readOnly = true)
  public List<LivroResponseDTO> buscarPorFaixaPreco(BigDecimal min, BigDecimal max) {
    if (min.compareTo(max) > 0) // compareTo: 1 = maior, 0 = igual, -1 = menor
      throw new BusinessException("Preço mínimo maior que o máximo");
    return livroRepository.findByPrecoBetween(min, max)
      .stream().map(this::paraResponseDTO).toList();
  }

  @Transactional(readOnly = true)
  public List<LivroResponseDTO> listarDisponiveis() {
    return livroRepository.findLivrosDisponiveis()
      .stream().map(this::paraResponseDTO).toList();
  }

  @Transactional
  public LivroResponseDTO atualizar(Long id, LivroRequestDTO dto) {
    Livro livro = livroRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Livro", "id", id));
    // Se outro livro tem o mesmo ISBN, é conflito
    if (livroRepository.existsByIsbnAndIdNot(dto.isbn(), id))
      throw new BusinessException("Outro livro já usa o ISBN: " + dto.isbn());
    Autor autor = autorRepository.findById(dto.autorId())
      .orElseThrow(() -> new ResourceNotFoundException("Autor", "id", dto.autorId()));
    livro.setTitulo(dto.titulo()); livro.setIsbn(dto.isbn());
    livro.setAnoPublicacao(dto.anoPublicacao()); livro.setPreco(dto.preco());
    livro.setEstoque(dto.estoque()); livro.setAutor(autor);
    return paraResponseDTO(livroRepository.save(livro));
  }

  @Transactional
  public LivroResponseDTO atualizarEstoque(Long id, AtualizarEstoqueDTO dto) {
    Livro livro = livroRepository.findById(id)
      .orElseThrow(() -> new ResourceNotFoundException("Livro", "id", id));
    livro.setEstoque(dto.estoque());
    return paraResponseDTO(livroRepository.save(livro));
  }

  @Transactional
  public void deletar(Long id) {
    if (!livroRepository.existsById(id))
      throw new ResourceNotFoundException("Livro", "id", id);
    livroRepository.deleteById(id);
  }

  private Livro paraEntidade(LivroRequestDTO dto, Autor autor) {
    return Livro.builder()
      .titulo(dto.titulo()).isbn(dto.isbn())
      .anoPublicacao(dto.anoPublicacao()).preco(dto.preco())
      .estoque(dto.estoque()).autor(autor).build();
  }

  public LivroResponseDTO paraResponseDTO(Livro l) {
    return new LivroResponseDTO(
      l.getId(), l.getTitulo(), l.getIsbn(), l.getAnoPublicacao(),
      l.getPreco(), l.getEstoque(),
      autorService.paraResumoDTO(l.getAutor()) // DTO aninhado do autor
    );
  }
}

Controllers — Endpoints HTTP
A porta de entrada da API: cada método é uma URL que o mundo pode chamar.
🧠 Conceito — REST e HTTP

HTTP é o protocolo que o browser e qualquer app usa para se comunicar. Tem "verbos" que indicam a intenção:

  • GET → ler/buscar dados (não modifica nada)
  • POST → criar um novo recurso
  • PUT → substituir um recurso por completo
  • PATCH → atualizar parte de um recurso
  • DELETE → remover um recurso

O Controller mapeia cada URL + verbo para um método Java.

package com.projeto.biblioteca.controller;

import com.projeto.biblioteca.dto.AutorDTO.*;
import com.projeto.biblioteca.service.AutorService;
import jakarta.validation.Valid;
import org.springframework.http.*;
import org.springframework.web.bind.annotation.*;
import java.util.List;

@RestController
// @RestController combina:
//   @Controller     = esta classe trata requisições HTTP
//   @ResponseBody   = o retorno dos métodos vai direto no corpo JSON
// Então ao retornar um objeto Java, o Spring automaticamente
// converte para JSON e manda para o cliente.

@RequestMapping("/autores")
// @RequestMapping define o PREFIXO de todas as rotas neste controller.
// Todos os endpoints começam com /autores

@CrossOrigin(origins = "*")
// @CrossOrigin permite que outros domínios (front-end, Postman)
// chamem esta API. origins = "*" = qualquer origem.

public class AutorController {

  private final AutorService autorService;

  public AutorController(AutorService autorService) {
    this.autorService = autorService;
  }

  // ── POST /autores ─────────────────────────────────────────
  // Cria um novo autor.
  // @PostMapping = aceita requisições HTTP POST em /autores
  @PostMapping
  public ResponseEntity<AutorResponseDTO> criar(
      @Valid @RequestBody AutorRequestDTO dto) {
    // @Valid = ativa as validações (@NotBlank, @Size...) do DTO.
    //         Se falhar, lança exceção → GlobalExceptionHandler captura → 400
    // @RequestBody = lê o corpo JSON da requisição e converte para AutorRequestDTO

    AutorResponseDTO resposta = autorService.criar(dto);
    return ResponseEntity
      .status(HttpStatus.CREATED) // HTTP 201 Created
      .body(resposta);
  }

  // ── GET /autores ou GET /autores?nome=machado ─────────────
  @GetMapping
  public ResponseEntity<List<AutorResponseDTO>> listar(
      @RequestParam(required = false) String nome) {
    // @RequestParam = lê parâmetro da URL: /autores?nome=machado
    // required = false = parâmetro é opcional. Se não informar, nome = null.

    if (nome != null && !nome.isBlank())
      return ResponseEntity.ok(autorService.buscarPorNome(nome));
    return ResponseEntity.ok(autorService.listarTodos()); // HTTP 200 OK
  }

  // ── GET /autores/1 ────────────────────────────────────────
  @GetMapping("/{id}")
  public ResponseEntity<AutorResponseDTO> buscarPorId(
      @PathVariable Long id) {
    // @PathVariable = extrai o {id} da URL.
    // GET /autores/3 → id = 3L
    return ResponseEntity.ok(autorService.buscarPorId(id));
  }

  // ── PUT /autores/1 ────────────────────────────────────────
  @PutMapping("/{id}")
  public ResponseEntity<AutorResponseDTO> atualizar(
      @PathVariable Long id,
      @Valid @RequestBody AutorRequestDTO dto) {
    return ResponseEntity.ok(autorService.atualizar(id, dto));
  }

  // ── DELETE /autores/1 ─────────────────────────────────────
  @DeleteMapping("/{id}")
  public ResponseEntity<Void> deletar(@PathVariable Long id) {
    autorService.deletar(id);
    return ResponseEntity.noContent().build();
    // HTTP 204 No Content = sucesso, sem corpo na resposta.
    // Faz sentido: o recurso foi deletado, não tem mais nada para retornar.
  }
}

package com.projeto.biblioteca.controller;

import com.projeto.biblioteca.dto.LivroDTO.*;
import com.projeto.biblioteca.service.LivroService;
import jakarta.validation.Valid;
import org.springframework.http.*;
import org.springframework.web.bind.annotation.*;
import java.math.BigDecimal;
import java.util.List;

@RestController
@RequestMapping("/livros")
@CrossOrigin(origins = "*")
public class LivroController {

  private final LivroService livroService;
  public LivroController(LivroService livroService) { this.livroService = livroService; }

  @PostMapping
  public ResponseEntity<LivroResponseDTO> criar(@Valid @RequestBody LivroRequestDTO dto) {
    return ResponseEntity.status(HttpStatus.CREATED).body(livroService.criar(dto));
  }

  // GET /livros ou GET /livros?titulo=dom ou GET /livros?precoMin=20&precoMax=50
  @GetMapping
  public ResponseEntity<List<LivroResponseDTO>> listar(
      @RequestParam(required = false) String titulo,
      @RequestParam(required = false) BigDecimal precoMin,
      @RequestParam(required = false) BigDecimal precoMax) {

    if (titulo != null && !titulo.isBlank())
      return ResponseEntity.ok(livroService.buscarPorTitulo(titulo));
    if (precoMin != null && precoMax != null)
      return ResponseEntity.ok(livroService.buscarPorFaixaPreco(precoMin, precoMax));
    return ResponseEntity.ok(livroService.listarTodos());
  }

  // ⚠️ IMPORTANTE: endpoints específicos ANTES do /{id}!
  // Se /disponiveis viesse depois, o Spring tentaria converter
  // "disponiveis" como um Long (id) e daria erro.
  @GetMapping("/disponiveis")
  public ResponseEntity<List<LivroResponseDTO>> disponiveis() {
    return ResponseEntity.ok(livroService.listarDisponiveis());
  }

  @GetMapping("/autor/{autorId}")
  public ResponseEntity<List<LivroResponseDTO>> porAutor(@PathVariable Long autorId) {
    return ResponseEntity.ok(livroService.listarPorAutor(autorId));
  }

  @GetMapping("/{id}")
  public ResponseEntity<LivroResponseDTO> buscarPorId(@PathVariable Long id) {
    return ResponseEntity.ok(livroService.buscarPorId(id));
  }

  @PutMapping("/{id}")
  public ResponseEntity<LivroResponseDTO> atualizar(
      @PathVariable Long id, @Valid @RequestBody LivroRequestDTO dto) {
    return ResponseEntity.ok(livroService.atualizar(id, dto));
  }

  // PATCH = atualização PARCIAL. Só atualiza o estoque,
  // sem precisar enviar todos os outros campos do livro.
  @PatchMapping("/{id}/estoque")
  public ResponseEntity<LivroResponseDTO> atualizarEstoque(
      @PathVariable Long id, @Valid @RequestBody AtualizarEstoqueDTO dto) {
    return ResponseEntity.ok(livroService.atualizarEstoque(id, dto));
  }

  @DeleteMapping("/{id}")
  public ResponseEntity<Void> deletar(@PathVariable Long id) {
    livroService.deletar(id);
    return ResponseEntity.noContent().build();
  }
}
